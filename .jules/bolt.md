## 2024-05-18 - Optimize relational data lookups in Local Store to avoid O(n²) bottleneck
**Learning:** Found a major performance bottleneck specific to this codebase's architecture in `src/lib/store.ts`. Relational data operations on the local JSON fallback store (`getJobs` merging scores, `getApplications` merging jobs and resumes) were heavily relying on `Array.find` within `Array.map` loops. This causes O(n²) complexity, which will result in massive lag as the number of user jobs, scores, applications, and resumes grow.
**Action:** Always pre-compute a `Map` dictionary for lookups before running the primary mapping loop when fetching relational items from the local JSON store. Converting O(n²) Nested Array loops to O(n) Map dict lookups provides measurable, reliable performance speed-ups in JavaScript for large arrays.
